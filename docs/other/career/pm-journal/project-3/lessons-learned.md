# Project 3 | Lessons Learned 

### Jira Timeline
First time utilizing the Jira Timeline instead of the timeline / gantt chart I created in Confluence

This worked very well. Was able to:
1. Plan over future sprints
1. Place User Storys (and their tasks) in priority order (top to bottom)
1. Represent User Story type tickets in the timeline
1. Represent the tasks for those User Stories by placing them under them and being able to plan for them in future sprints

<br>

### "Story" type tickets
Represented every user story in a "Story" type Jira ticket.  

<br>

### High level estimatng for User Stories
If a User Story isn't broken down into tickets yet then applying a highlevel estimate (using dot system) worked really well for planning

NOTE: Remove this estimate when the user story is decomposed

<br>

### Decompose User Story into Tasks and other approaches
When we were ready to decompose a user story I utilized the "Task" type Jira ticket. Also the following was done: 
1. Used nomenclature to create relationship between the task and it's user story. (e.g. US 11 | Theme buttons)
1. Utilized the "relationship" option in Jira to think the user story and the task. I normally used "Resolves"
1. In the timeline I positioned tasks directly under the story ticket so you could visually see how many tasks were in the story and also could plan easier
